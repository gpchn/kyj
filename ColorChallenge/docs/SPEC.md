# ColorChallenge (色差挑战) - Creative Specification

## 1. 概念与愿景

### 背景阐述
色谱采集是将色彩理论转化为游戏体验的教育性艺术作品。色彩是视觉艺术的基础语言，理解色彩的和谐与对比是每个艺术创作者的必修课。本作品将抽象的色彩构成知识转化为直观的游戏机制，让玩家在"补全画面"的过程中，自然而然地掌握色彩搭配的奥秘。

### 创作目标
- 将色彩构成课程游戏化
- 从简单到复杂的难度递进设计
- 培养玩家对色彩的敏感度
- 创造寓教于乐的学习体验

### 艺术方向
以"抽象画"为视觉载体，每一关都是一幅待完成的抽象艺术作品。玩家选择的颜色直接影响画面的最终效果，游戏过程本身就是一次艺术创作。色彩选择不仅是正确与否的判断，更是审美偏好的表达。

## 2. 设计语言

### 美学基础
- **核心风格**: 抽象画风格，色块构成
- **质感追求**: 平面色块，边缘清晰
- **色彩哲学**: 色彩理论为核心

### 色彩系统
| 类型 | 说明 | 示例 |
|------|------|------|
| 互补色 | 色轮上相对的颜色 | 红-绿、蓝-橙 |
| 邻近色 | 色轮上相邻的颜色 | 红-橙-黄 |
| 三角色 | 色轮上等距的三色 | 红-黄-蓝 |
| 分裂互补 | 互补色两侧的颜色 | 红-黄绿-蓝绿 |

### 字体选择
- **主标题**: "Poppins" (Google Fonts) - 现代几何
- **辅助文字**: "Inter" - 清晰易读

### 空间系统
- 左侧：抽象画作展示区
- 右侧：色轮选择器
- 底部：关卡进度指示

### 动效哲学
- 色块填充采用平滑过渡
- 正确选择时画面闪烁庆祝
- 错误选择时轻微抖动提示

## 4. 功能与交互

### 核心功能

#### 4.1 关卡系统
- **难度递进**: 从简单互补色到复杂邻近色对比
- **关卡数量**: 共 20 关
- **解锁机制**: 完成前一关解锁下一关

#### 4.2 色轮选择器
- **类型**: HSL 色轮
- **交互**: 点击选择色相，拖动调整饱和度
- **预览**: 实时预览选择效果

#### 4.3 评分系统
- **精确度**: 根据选择颜色与目标颜色的差异评分
- **等级**: S/A/B/C 四个等级
- **反馈**: 详细说明为何是这个颜色

### 交互细节
| 交互 | 视觉反馈 |
|------|----------|
| 点击色轮 | 显示选中颜色预览 |
| 确认选择 | 填充色块 + 评分动画 |
| 正确选择 | 画面闪烁 + 星星动画 |
| 错误选择 | 轻微抖动 + 提示信息 |

### 边界情况
- 色盲模式：提供图案辅助
- 跳过关卡：允许跳过但无评分
- 重新挑战：可随时重玩已通关关卡

## 5. 组件清单

### 5.1 抽象画作组件 (AbstractPainting)
- **构成**: 多个色块组合
- **状态**: 缺失一块
- **动画**: 填充过渡动画

### 5.2 色轮选择器 (ColorWheel)
- **类型**: HSL 圆形色轮
- **交互**: 点击选择
- **辅助**: 显示当前选中颜色

### 5.3 评分面板 (ScorePanel)
- **内容**: 等级 + 精确度百分比
- **动画**: 弹出显示
- **按钮**: 下一关 / 重玩

### 5.4 关卡指示器 (LevelIndicator)
- **样式**: 圆点序列
- **状态**: 完成、当前、锁定
- **交互**: 点击跳转已完成关卡

### 5.5 色彩提示 (ColorHint)
- **功能**: 显示色彩理论提示
- **触发**: 点击问号按钮
- **内容**: 互补色、邻近色等概念解释

## 6. 技术实现

### 框架选择
- **渲染**: HTML5 Canvas 2D + CSS
- **色轮**: SVG 或 Canvas 绘制
- **动画**: CSS Animations + requestAnimationFrame

### 性能目标
- 60fps 流畅渲染
- 关卡加载: < 500ms
- 色轮响应: 即时
